Clases y Objetos

Una clase es una plantilla que define un conjunto de características y comportamientos que pueden tener los objetos que se derivan de ella. Se considera el esquema básico a partir del cual se pueden crear múltiples instancias, conocidas como objetos. Cada objeto tiene sus propios valores específicos para los atributos definidos en la clase. Por ejemplo, si consideramos una clase que representa un "Automóvil", esta puede contener atributos como el color, la marca y el modelo, además de métodos que permitan realizar acciones como acelerar o frenar. Cuando se crea un objeto basado en esta clase, se generan instancias que tienen valores particulares para esos atributos.

Pilas y colas

Las estructuras de datos son fundamentales en la programación, ya que permiten organizar y manipular la información de manera eficiente. Entre ellas, las pilas y colas son estructuras lineales utilizadas en múltiples aplicaciones, desde la gestión de memoria hasta la implementación de algoritmos en inteligencia artificial. Las pilas siguen el principio LIFO (Last In, First Out), lo que significa que el último elemento en entrar es el primero en salir. Por otro lado, las colas siguen el principio FIFO (First In, First Out), donde el primer elemento en entrar es el primero en salir. Ambas estructuras se utilizan en diferentes aplicaciones como gestión de procesos, almacenamiento temporal de datos y algoritmos computacionales.

  • Pilas
    1. push(): Añadir un elemento a la pila.
    2. pop(): Eliminar y devolver el último elemento agregado.
  • Colas
    1. enqueue(): Agregar un elemento al final de la cola.
    2. pdequeue(): Eliminar y devolver el primer elemento agregado.
